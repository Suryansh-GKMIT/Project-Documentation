Use Case Document

Project: SkillCheck – Enclosed Access Online Test Platform (MVP)

1. Introduction

SkillCheck is an internal online test platform designed for enclosed environments such as organizations, institutions, or training departments. The system includes three roles: Admin, Test Creator, and Test Taker. The Admin manages all users and permissions. Test Creators design tests and assign them to specific users. Test Takers log in to view and attempt the tests assigned to them. The MVP focuses on time-based assessments without proctoring features.

2. Actors / Roles

2.1 Admin

Responsible for managing platform users, assigning permissions, and overseeing tests and results.

2.2 Test Creator

Authorized user who can create tests, add questions, assign Test Takers, and view results.

2.3 Test Taker

End user who logs in to attempt tests assigned to them.

3. Preconditions

  1. Only Admin can create or authorize user accounts.
  2. Admin must explicitly assign the Test Creator role before a user can create tests.
  3. Test Takers must be registered users.
  4. Test Creators must assign Test Takers before tests become visible to them.
  5. Tests must contain at least one question before being published.
  6. Published tests become available only to assigned users.

4. Use Cases

4.1 Admin Adds and Manages Users

Actor: Admin
Preconditions: Admin is logged in.
Main Flow: 1. Admin opens User Management. 2. Admin adds new users or updates existing users. 3. Admin grants or revokes Test Creator permissions. 4. Admin activates, deactivates, or removes users. Postcondition: User accounts and permissions updated. Exceptions: Invalid data or database conflicts.

4.2 Test Creator Login

Actor: Test Creator
Preconditions: Creator permissions granted by Admin.
Main Flow: 1. Creator logs in. 2. System verifies role. 3. Creator dashboard loads. Postcondition: Creator can access test creation tools. Exceptions: Unauthorized users receive an “Access Denied” message.

4.3 Create Test

Actor: Test Creator
Preconditions: Creator is logged in.
Main Flow: 1. Creator clicks “Create Test.” 2. Enters title, description, and test duration. 3. Saves the test in Draft state. Postcondition: Test saved with Draft status. Exceptions: Missing required fields.

4.4 Add or Edit Questions

Actor: Test Creator
Preconditions: Test exists in Draft state.
Main Flow: 1. Creator selects the test. 2. Adds questions with text, options, and one correct answer. 3. Optionally edits or deletes questions before publishing. 4. Saves the updates. Postcondition: Questions linked to the test. Exceptions: Missing text or correct answer.

4.5 Assign Users to Test

Actor: Test Creator
Preconditions: Test exists and users exist in system.
Main Flow: 1. Creator opens “Assign Users” panel. 2. Selects users from the registered user list. 3. Assigns users to the test. 4. Saves assignments. Postcondition: Users assigned to the test. Exceptions: No users selected or database issues.

4.6 Publish Test

Actor: Test Creator
Preconditions: Test must have at least one question.
Main Flow: 1. Creator marks test as Active. 2. System approves state change. Postcondition: Test becomes visible to assigned Test Takers. Exceptions: Attempt to publish without any questions.

4.7 Test Taker Attempts Test

Actor: Test Taker
Preconditions: Test assigned to the Test Taker and set to Active.
Main Flow: 1. Test Taker logs in. 2. Navigates to Assigned Tests. 3. Selects the test and clicks “Start.” 4. Timer begins. 5. User answers questions. 6. Submits test manually or system auto-submits on timeout. Postcondition: Responses stored and scored. Exceptions: Test already attempted or expired.

4.8 View Results

Actors: Test Creator, Admin
Preconditions: At least one test attempt exists.
Main Flow: 1. User opens Results section. 2. Selects a test. 3. Views list of attempts. 4. Views detailed responses and score. 5. Optionally exports CSV. Postcondition: Results displayed and available for export. Exceptions: No attempts found.

5. Future Scope Use Cases

  1. Tab-switch and window-switch monitoring.
  2. Camera and microphone requirements.
  3. Face detection and cheating flagging.
  4. Live proctoring dashboard.
  5. Mobile application.
  6. Randomized question order per attempt.
